# control-tower

## What (これは何か？)

**Control Tower** は、あなたの創作活動全体を統括する「司令塔」となるリポジトリです。

様々な種類のプロジェクト（物語、イラスト、技術ブログなど）を、それぞれの目的に特化したテンプレートから効率的に生成するための、自動化スクリプト群を管理・実行する中心的な場所（ハブ）です。

## Why (なぜ作るのか？)

`idea-generator` や `story-forge-template` のように、目的別のリポジトリが増えてくると、それらを横断的に管理・利用する仕組みが必要になります。

この `control-tower` は、各ツールを連携させ、創作活動の開始をワンコマンドで自動化するために作ります。「新しい物語を始めたい」と思ったら、ここからスクリプトを実行するだけで、`story-forge-template` を基にした新しいプロジェクトが即座に立ち上がる、というような体験を目指します。

## For Whom (誰のためか？)

- 複数の創作プロジェクトやツールを管理している、未来の自分自身。
- 創作の初期設定にかかる時間を限りなくゼロに近づけ、アイデアの具現化に集中したいすべてのクリエイター。

## Usage (使い方)

`ct` は、あなたの創作活動を開始するためのコマンドラインツールです。
どのディレクトリからでも実行できます。

### 1. 対話モード (Interactive Mode)

コマンドを覚えなくても、メニューから選択するだけでプロジェクトを作成できます。

```bash
ct
```

実行すると、利用可能なプロジェクトタイプのメニューが表示されます。

### 2. コマンドモード (Command Mode)

作成したいプロジェクトタイプがわかっている場合は、直接コマンドで指定できます。

```bash
ct new <type> <project-name>
```

- `<type>`: 作成したいプロジェクトの種類 (例: `story`)。これは `scripts/create-<type>.sh` に対応します。
- `<project-name>`: 新しいプロジェクトのディレクトリ名。

**例:**

```bash
ct new story my-epic-fantasy
```

## How to Extend (拡張方法)

`control-tower` の能力を拡張するのは簡単です。

1. `scripts/` ディレクトリに `create-your-type.sh` という命名規則で新しいシェルスクリプトを追加します。
2. スクリプトに実行権限を付与します (`chmod +x scripts/create-your-type.sh`)。

これだけで、`ct` コマンドは新しいプロジェクトタイプを自動的に認識し、対話モードとコマンドモードの両方で利用可能になります。
